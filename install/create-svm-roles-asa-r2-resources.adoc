= ASA R2システム用のSVMロールの作成
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ASA r2 システムで新しい SVM ユーザーのロールを作成するには、いくつかのONTAP CLI コマンドを実行する必要があります。このロールは、 ASA r2 システムで SVM をSnapCenterで使用するように構成し、vsadmin ロールを使用しない場合に必要です。

* 手順 *

. ストレージシステムで、ロールを作成してすべての権限を割り当てます。
+
`security login role create –vserver <svm_name\>- role <SVM_Role_Name\> -cmddirname <permission\>`

+

NOTE: このコマンドは権限ごとに繰り返す必要があります。

. ユーザを作成し、そのユーザにロールを割り当てます。
+
`security login create -user <user_name\> -vserver <svm_name\> -application http -authmethod password -role <SVM_Role_Name\>`

. ユーザのロックを解除します。
+
`security login unlock -user <user_name\> -vserver <svm_name\>`
